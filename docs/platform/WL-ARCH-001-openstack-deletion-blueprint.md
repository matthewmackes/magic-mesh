## Area: mackesd OpenStack WORKER/BACKEND tree (crates/mesh/mackesd/src/workers/openstack/ + mackes_mesh_types::openstack)

The OpenStack backend is two trees: (1) the mackesd `openstack` worker dir — 22 .rs files (~17,652 LOC) + 15 JSON contract fixtures — a Kolla-service Podman supervisor with a Nova/Heat/Keystone/Glance/Cinder/Swift/Designate feature stack, an `action/cloud/*` verb responder, an IaC HTTP client, and a `state/openstack/<node>` mirror; and (2) the shared `mackes_mesh_types::openstack` types module (1697 LOC) that the neutral `mackes_mesh_types::cloud` facade currently RE-EXPORTS from. The delete is safe to do immediately for compile IF: the `cloud` facade first absorbs (owns) the neutral type/fn bodies instead of re-exporting them, then every consumer rewires to `mackes_mesh_types::cloud::*`. Hard compile breakers on delete: the cloud facade (cloud.rs re-export + 2 direct calls), the shell IaC/cloud_plane surfaces in crates/desktop (5 sites — these break the WORKSPACE build), session_broker's QC-14 Nova/Heat placement path (imports HOT_TEMPLATE_VERSION + verbs; first_desktop chains onto it), kdc_host's KDC-MESH-8 cloud-lifecycle commands (imports CloudInstance/CloudReply/LifecycleAction/cloud_action_topic), the spawn site, the workers::mod + lib.rs module decls, and the worker_role census entry (6 numeric asserts + 1 dedicated test + several census-message strings). Decoupled (NO compile break, Bus-JSON only): unit_aggregator's `state/openstack/<node>` consumer, the chat worker's `event/notify/` fold, and the shell explorer's local `CLOUD_ACTION_PREFIX` mirror const. Only OpenStack-exclusive Cargo dep is `serde_yaml` (clouds.yaml parse); reqwest/base64/rand are shared and must stay. No mackesd clap subcommand is tied to OpenStack — the "CLI" surface is the 18 `action/cloud/*` Bus verbs (CLOUD_VERBS).
